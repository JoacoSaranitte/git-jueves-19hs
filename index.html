Sistema de control de versiones

Areas de Git

Working Directory

Es la carpeta en la que iniciamos el repositorio, donde realizamos los cambios en los archivos y muy rara vez estaremos utilizando la consola.

Staging Area (INDEX)

Es el area de control de cambios, un archivo que identifica y verifica las modificaciones en el directorio de trabajo. Todo archivo al que querramos hacerle seguimiento debe ser agregado previamente al INDEX para que realice su snapshot

Repository (Local / Remote)

Es el area donde almacenan los cambios confirmados en formato BLOB (Binary Object File). Esto nos permite reestablecer tanto archivos, directorios, o hasta el mismo proyecto utilizando como referencia dichos archivos e indicando a git en que instancia del mismo.
